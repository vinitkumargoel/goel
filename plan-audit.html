B. Transport linchpin (headline finding)

Load-bearing false assumption in v2: “bind an adapter → traffic egresses that WAN.” Only conditionally true — and not true for plain CURLOPT_INTERFACE on macOS.

Routing physics

Source-IP bind() alone: socket bound to en0's IP kernel routes by destination packet may leave via en1 (default route) wire sees src=en0_IP on en1 → BCP38 / RPF drop at ISP Interface-scoped socket (required): setsockopt IP_BOUND_IF / SO_BINDTODEVICE → packets forced out that interface

Platform reality

Platformcurl CURLOPT_INTERFACE todayReal egress control
macOS Name→IP then plain source bind. No IP_BOUND_IF in curl non-Linux path. Fails flagship non-primary NIC (hotspot beside Wi‑Fi)
Linux Name → SO_BINDTODEVICE. IP alone → source-bind only. Works with name; plan’s “prefer IP” was wrong

The fix (keeps curl, ~MVP effort)

CURLOPT_SOCKOPTFUNCTION after socket(), before connect(): macOS: ifindex = if_nametoindex(name) setsockopt(fd, IPPROTO_IP, IP_BOUND_IF, &ifindex) setsockopt(fd, IPPROTO_IPV6, IPV6_BOUND_IF, &ifindex) Linux: setsockopt(fd, SOL_SOCKET, SO_BINDTODEVICE, name, …) Phase-0 must prove: · nettop/interface counters show bytes on non-primary NIC · echo service shows distinct public IPs when paths truly diverge · control: plain CURLOPT_INTERFACE fails non-primary (documents why)

Also correct in research

URLSession no bind · MPTCP insufficient · H3 off defensible as conservative · aria2 --interface is global single-NIC, not multi-segment fan-out.

Fallback if spike fails

Network.framework requiredInterface is reliable on macOS but implies HTTP client rewrite (“very high”) — only if SOCKOPT path fails.

Refs: curl lib/cf-socket.c bindlocal; Apple DTS forums; curl issues 13613/11599; Linux SO_BINDTODEVICE unprivileged since 5.7; codingrelic / networkop on source-bind vs egress.

C. Gaps the original plan missed (now in v3)

  1. RateLimiter × curl — per-handle MAX_RECV_SPEED ⇒ N× cap. Fix: curl cap 0, Swift pace only.
  2. HTTP/2 — multiplex = one interface. Fix: ≥1 distinct connection per adapter; no cross-adapter reuse.
  3. Cookie domain parity — Foundation scopes per domain automatically; curl jar must replicate deliberately.
  4. Blocking bridge threads — adapters × streams × tasks OS threads; Sendable callback boundary; check_sendable in repo.
  5. CGNAT double edge — same public IP on “two” WANs kills gain and multi-client fear; more relevant than port exhaustion.
  6. Linux expensive signal missing — no NWPathMonitor on daemon; consent story incomplete without heuristics/manual flag.
  7. Primary Content-Range total — not already handled; multi-path on primary needs new validation.
  8. rp_filter / policy routing — only if someone falls back to source-IP bind (not MVP); was absent from v2.
