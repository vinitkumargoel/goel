Network Aggregation

Combine multiple internet adapters so one file fans out across them, with live UI showing which adapter each connection uses. This revision incorporates the deep code + networking audit: the MVP transport is no longer plain CURLOPT_INTERFACE — it is egress-scoped socket bind via CURLOPT_SOCKOPTFUNCTION + platform APIs.

Verdict · Feasible if egress spike passes MVP transport · SOCKOPT + IP_BOUND_IF / SO_BINDTODEVICE Top risk · macOS non-primary NIC egress Companion · plan-audit.html

Headline change from v2: source-IP bind ≠ multi-WAN

Kernel routes by destination. A plain source-address bind (CURLOPT_INTERFACE on macOS today) does not force packets out a non-primary interface — and may even produce BCP38-dropped packets (src IP from en0, egress en1). Flagship scenario “Wi‑Fi + phone hotspot” requires interface-scoped sockets: macOS IP_BOUND_IF / IPV6_BOUND_IF, Linux SO_BINDTODEVICE (name, not IP). Phase‑0 must prove per-interface egress, not mere bind success.

v3 changelog (from deep audit)

  • Transport decision rewritten: SOCKOPTFUNCTION + platform bind-if, not source-IP curl interface alone.
  • Phase‑0 gate #1 = non-primary NIC bytes in nettop + distinct public IPs where paths diverge.
  • Expensive-path story corrected: pause is opt-in default-off + macOS-only; multi-path needs independent gate; Linux classifier gap noted.
  • Primary Content-Range total validation elevated to new MVP task (mirror-only today).
  • Removed “prefer bind to local IP” and CAP_NET_RAW as primary Linux risk; bind by name/index.

  • Governor rebalance marked new, not a reuse (current governor is monotonic-decreasing).

0. Executive summary

Doable

What works

HTTP range segments already fan out. Pin each segment’s socket to a NIC with real egress scoping. Label adapters in Connections UI.

Hard (corrected)

What is hard

macOS curl cannot IP_BOUND_IF today — we must install SOCKOPTFUNCTION. Linux name≠IP. Dual stack HTTP parity, cookies, rate limit aggregate.

Not this

What it is not

Not LACP, not Speedify bonding, not plain bind(sourceIP), not aria2-style global single-interface, not guaranteed 2×.

Definition of done unchanged in spirit, stricter in verification: Wi‑Fi (primary) + phone hotspot (non-primary) must show bytes on both interfaces in nettop / interface counters — not just “socket bound successfully.”

2. Codebase reality (audit-verified)

In our favor (verified line-for-line)

  • SegmentedTransfer multi-range fan-out, preallocate, incomplete-segment guards (per-segment + aggregate).
  • Resume refuses without ETag/Last-Modified validators — stronger than many clients.
  • 206-only for ranged; full 200 → reject. Governor shrinks on 429 (monotonic decrease).
  • RateLimiter is per-download aggregate across segments — must preserve this with curl.
  • RedirectSanitizer allow-list strips Auth/Cookie/Referer/custom on host change or https→http.
  • TaskConnection + Connections tab + remote ConnRow exist; resume cursor has no adapter identity.
  • CurlBridge FTP-only today; libcurl already linked both platforms; blocking by design.

3. Network physics + routing truth

WRONG mental model (v2 risk) socket.bind(en0_IP) → kernel still routes by destination → packet may leave via en1 (default route) → src=en0_IP on en1 wire → ISP BCP38 drop RIGHT model (v3) socket() → IP_BOUND_IF / SO_BINDTODEVICE(ifindex|name) → connect() → packets forced out that interface's L2/L3 path Case A — real multi-WAN en0 Wi‑Fi → Router A → ISP fiber ──┐ en1 USB tether → Phone → cellular ──┼── Internet ⇒ independent last-mile; public IPs often differ Case B — multi-NIC same gateway en0+en1 → same home router → one WAN ⇒ no speedup (honest UX / same-public-IP probe) Case C — VPN full tunnel Default route utun0; physical bind may bypass VPN (security) ⇒ forbid physical multi-path unless scary override
CGNAT double edge: two “different” WANs on the same mobile carrier can share one public IP — undermines throughput premise and reduces multi-client 429 risk. More important than “port exhaustion” (dozens of conns ≪ typical CGNAT port pools).

aria2 note: --interface is a global single-NIC bind for all sockets — not a reference for per-segment multi-interface fan-out. That logic is ours to build.

4.3 HTTP parity checklist (curl path)

  • Range + 206-only + Content-Range total on every response (primary included)
  • Redirect sanitizer allow-list parity (host change / https→http)
  • UA, Referer, extra headers, Basic auth TLS-only
  • Cookie jar with domain scoping like Foundation (not a naive shared bag)
  • Proxy: system / manual HTTP / SOCKS5-as-manual-subtype / none — aggregation inactive under manual proxy (v1)
  • Linux proxy today = env vars (http_proxy), not CFNetwork dictionary
  • Cancel on pause; progress → resume cursor cadence; stall detection
  • Swift RateLimiter.pace in write path; curl per-handle speed cap = 0

4.4 Linux extras for spike

  • Confirm unprivileged SO_BINDTODEVICE on target kernels (5.7+ relaxed CAP_NET_RAW need for unbound sockets).
  • If ever testing source-IP-only fallback: need policy routing + rp_filter — not MVP path.
  • Include Linux in Phase‑0; do not defer “does egress work?” to polish.

6. Edge cases (condensed + audit additions)

Egress / bind / OS
  • Source-IP bind without bind-if → wrong egress / BCP38 drop.
  • Linux IP-only CURLOPT_INTERFACE → no SO_BINDTODEVICE.
  • Non-primary Service Order interface (hotspot) — must be index-scoped.
  • VPN full tunnel vs split tunnel destination failures.
  • CGNAT same public IP on “two” WANs.
  • Virtual ifaces: utun, awdl, llw, bridge, ap — hide by default.
  • Hotplug USB Ethernet / iPhone tether mid-session.
HTTP (MVP)
  • NEW: validate Content-Range total on primary + every adapter (not just mirrors).
  • No ranges / unknown size → single stream on preferred adapter.
  • Resume cursor: ranges+validators only — never adapter IDs.
  • H2: distinct conn per adapter; multiplex OK within one adapter.
  • H3 off in MVP.
  • RateLimiter Swift-side only; curl speed cap off.
  • Cookies: domain-scoped like Foundation.
  • Signed URL expiry on slow path; captcha HTML ≠ success.
  • Mirrors: adapters primary, mirrors failover only.
Scheduling / multi-task / concurrency
  • Low profile enableExtraConnections=false → no multi-path override.
  • Global connection budgets still apply across multi-path tasks.
  • Governor monotonic — weighted rebalance is Phase‑3 new code.
  • Auto-retry: after multi-path fail streak, try single-path once.
  • Settings / adapter set changes → apply on next resume (v1).
  • Blocking threads: adapters × streams × concurrent tasks — hard cap streamsPerAdapter.
Expensive / Linux / consent
  • Independent multi-path expensive gate (default off), separate from pause-on-expensive.
  • macOS: can use NWPathMonitor isExpensive for classification hints.
  • Linux: no NWPathMonitor — name heuristics / user marks adapter expensive / always confirm.

8. Security & privacy

  • RedirectSanitizer allow-list on every curl redirect — P0 tests.
  • VPN bypass default forbid when default route is VPN.
  • Expensive consent independent gate + counter; Linux no automatic detector.
  • Public-IP probe: stricter than NetworkGuard (block LAN + link-local); opt-in; privacy copy (“server may see multiple client IPs”).
  • No LAN scanning — enumerate local NICs only.

11. Test plan

Phase‑0 (blocking)

  1. macOS: Wi‑Fi primary + USB hotspot secondary — SOCKOPT IP_BOUND_IF — nettop shows bytes on both; echo shows distinct public IPs when paths diverge.
  2. macOS control: plain CURLOPT_INTERFACE only — document failure on non-primary (proves why SOCKOPT is required).
  3. Linux: name-based SO_BINDTODEVICE works unprivileged; IP-only does not steer.
  4. VPN present: physical bind behavior documented; default product policy = off.

Unit / integration

  • Adapter filters; pool demotion; resume cursor adapter-free; primary Content-Range validation.
  • Redirect sanitizer on curl redirects; cookie domain scoping; RateLimiter aggregate under N handles.
  • H2/isolation: two adapters ⇒ two connections (no cross reuse).
  • Policy: proxy / VPN / Low profile / expensive gate matrix.
  • Chaos: unplug secondary; flapping; pause/remove cancel (no zombie writers).

14. Kill / pivot criteria

Kill multi-path

Stop

SOCKOPT+IP_BOUND_IF cannot make non-primary NIC carry download bytes on release builds · VPN-safe design impossible · dual stacks unmaintainable · dual-WAN lab shows no gain when egress works.

Pivot

Alternatives

Per-download single-adapter pin only · Network.framework HTTP rewrite · external bonding · “path preference” without fan-out.

MVP done when: Wi‑Fi + hotspot both carry ranged bytes (nettop truth) · Connections labels match · unplug hotspot still finishes · expensive off · VPN not bypassed by default · same-router users get honest tip · RateLimiter still aggregate · feature flag can kill engine path instantly.
